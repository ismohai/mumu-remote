# MuMu Remote 快速上手（当前实现）

## 1. 现在支持的核心流程

- PC 端界面：
  - 只有一个主控按钮：`启动控制` / `关闭控制`
  - 启动后显示二维码（手机扫码配对）
  - 有 `配对列表`（已配对设备）
  - 有 `可配对设备（自动搜索）`，可点击发起配对
- 手机 App：
  - 主界面只有 `开始配对` 按钮
  - 打开 App 即自动进入被发现状态（不点按钮也能被 PC 搜索到）
  - 支持扫码配对
  - 支持被 PC 发起配对时弹窗确认

## 2. 前置准备

### Windows PC

1. 安装 Rust（`cargo`）
2. 安装 ADB，并保证命令行能执行 `adb`
3. 安装并启动 MuMu 模拟器
4. 确认 MuMu ADB 目标地址（默认 `127.0.0.1:7555`）

如果不是默认端口，启动前设置环境变量：

```powershell
$env:MUMU_ADB_TARGET="127.0.0.1:7555"
```

### Android

1. 安装 Android Studio
2. 手机和 PC 在同一局域网

## 3. 启动 PC

```powershell
cd pc/mumu-remote
cargo run
```

操作：

1. 点击 `启动控制`
2. PC 显示二维码
3. 使用手机配对

## 4. 手机配对方式

### 方式 A：手机扫码（推荐）

1. 打开手机 App，点 `开始配对`
2. 扫 PC 二维码
3. PC 会弹出确认框
4. PC 点击允许后完成配对并可进入控制

### 方式 B：PC 发起配对

1. 手机 App 打开后不做任何操作（保持前台即可）
2. PC 在 `可配对设备（自动搜索）` 里找到手机
3. 点击 `发起配对`
4. 手机弹出确认框，点击允许
5. 完成配对

## 5. 控制与画面

- 配对成功后，PC 会向已选配对设备推送画面
- 手机端支持触控转发、右侧滑出导航（返回/主页/多任务/设置/断开）

## 6. 常见问题

1. **PC 中文乱码**
   - 当前版本已在 Windows 上自动加载中文字体（`simhei/msyh/simsun`）
   - 若系统缺失这些字体，安装任一中文字体后重启程序

2. **手机没被自动发现**
   - 确认手机和 PC 在同一网段
   - 确认手机 App 已打开（主界面即可）
   - 确认防火墙放行 UDP `56000`

3. **有画面但控制无效**
   - 检查 PC 端 `adb connect` 是否成功
   - 手动验证：

```powershell
adb connect 127.0.0.1:7555
adb -s 127.0.0.1:7555 shell input keyevent 3
```

## 7. 说明

- 当前画面链路为 JPEG 帧流（可用版），不是最终 H.264 硬编优化版
- Android 端因当前环境无 `gradle` 可执行文件，尚未在本机做完整构建验证
