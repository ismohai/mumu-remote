# MuMu 局域网远控系统设计

## 1. 目标

- 在局域网内，用安卓手机以极低延迟控制 Windows 上的 MuMu 模拟器玩游戏。
- 画面目标：2K 分辨率、90fps（在网络与硬件允许的情况下尽量接近）。
- 操作体验：接近真机，支持多点触控、滑动、长按等常见手势，不依赖按键映射。
- 不传输声音，手机端只负责显示画面和控制输入。
- 只适配单实例 MuMu（一次控制一个模拟器窗口）。

非目标：
- 不支持公网/穿透，只面向同一局域网。
- 不支持 iOS 客户端。
- 暂不兼容其他模拟器（雷电、BlueStacks 等）。

## 2. 整体架构

- 形态：一台 Windows10 PC（MuMu + Rust 后台服务） + 一台安卓手机（客户端 App）。
- 通信方式：局域网 UDP 为主，必要时辅以 TCP/HTTP 控制通道（用于配对、状态查询等）。
- 数据链路：
  1. 画面链路：MuMu 窗口 → Windows 截屏/窗口捕获（DXGI / Win32）→ GPU 硬件编码 H.264/HEVC → UDP 发送视频帧 → 安卓端 `MediaCodec` 硬解码 → SurfaceView/TextureView 渲染。
  2. 输入链路：安卓多点触控/边缘手势 → 自定义输入事件协议（UDP 或可靠模式）→ Windows Rust 服务 → 转换为 adb 触控/按键指令 → 注入 MuMu 内部 Android 系统。
- 控制管理：
  - Rust 服务负责：MuMu 实例发现与绑定、分辨率查询设置、编码参数调整、连接管理、配对与认证。
  - 安卓 App 负责：连接/断开、右侧导航栏 UI、手势解析、全局设置（分辨率/刷新率等）。

## 3. Windows 端（Rust 服务）设计

### 3.1 进程形态

- 独立桌面应用（带最小化到托盘的 UI），用于：
  - 启动/停止“远控服务”。
  - 显示当前 MuMu 绑定状态（找到哪个窗口）。
  - 显示二维码用于手机首次配对。
  - 显示连接状态（已连接手机、帧率、分辨率等）。
- 后台自动维持一个局域网监听服务（固定端口或配置端口）。

### 3.2 MuMu 发现与绑定

- 仅支持 MuMu：
  - 通过窗口标题、进程名或类名查找 MuMu 窗口（例如窗口名包含 "MuMu"，具体到实际安装后再调试）。
  - 或者允许用户在桌面应用里点击按钮，让程序列出候选窗口让用户选择一次记住。
- 绑定策略：
  - 只绑定一个前台 MuMu 窗口。
  - 如果 MuMu 被关闭或窗口句柄无效，自动停止采集并通知手机端。

### 3.3 画面采集与编码

- 采集：
  - 优先使用 DXGI Desktop Duplication 或类似 API 对指定窗口区域进行采集（按窗口坐标裁切），避免采整屏再剪裁的额外开销。
  - 帧率控制：目标 90fps，采用定时器或基于垂直同步的方式驱动采集循环，允许动态降帧以匹配编码/网络能力。
- 编码：
  - 使用 Windows 下可用的硬件编码接口（如 Media Foundation + 硬件 MFT，或调用 ffmpeg/extern 工具库 封装硬编），封装在 Rust 中。
  - 支持 H.264（主用）和 HEVC（可选）两种编码模式，优先 H.264 以兼容更多安卓设备。
  - 码率策略：
    - 初始根据分辨率和帧率估算一个合理码率（例如 2K@90fps 起步 15–25Mbps），之后可根据丢包率和解码情况动态微调。
  - 延迟优化：
    - 减少编码缓冲（低 B 帧、低 lookahead）。
    - 发送端仅保持 0–1 帧队列，避免堆积。

### 3.4 传输协议

- 采用自定义 UDP 协议：
  - 视频帧拆分为多个 datagram，每个包包含：
    - 会话 ID
    - 帧序号
    - 分片序号/总分片数
    - 时间戳（发送时间）
  - 基本丢包容忍：
    - 小部分分片缺失时，允许丢帧（提高实时性）。
    - 将来可以考虑简单 FEC，但初版先不做。
- 控制/管理可以使用：
  - 一个轻量 TCP 或 UDP 可靠层，用于：
    - 握手、认证（基于配对 token）。
    - 分辨率/刷新率设置请求。
    - 状态查询（当前分辨率、帧率、网络状况）。

### 3.5 输入处理（adb）

- 与 MuMu 的交互：
  - 本机通过 adb 与 MuMu 内部 Android 系统通信（一般是模拟器预置的 adb 监听端口）。
  - Rust 封装 adb 命令/协议，将来自手机的指令转成：
    - 多点触控（touch down/move/up，带 pointer id）。
    - 常规按键：返回、主页、多任务等。
- 坐标映射：
  - 根据当前 MuMu 内部分辨率，将手机屏幕坐标映射到模拟器坐标。
  - 支持“跟随模拟器分辨率”：
    - 启动时通过 adb 读取模拟器分辨率。
    - 当用户在 MuMu 内更改分辨率时，可以在一定间隔重新读取并同步到手机端显示。

### 3.6 配对与认证

- 首次配对流程：
  - Rust 服务生成：
    - 一次性配对 token（随机字符串）。
    - 本机局域网 IP + 端口信息。
  - 将上述信息编码成二维码在桌面端展示。
  - 手机扫描后，向指定 IP/端口发起配对请求，携带 token。
  - 验证成功后，将该设备标记为“已配对”，保存一个长期设备 ID。
- 后续连接：
  - 已配对的手机可以在 App 内直接选择该电脑并连接，无需再次扫码。
  - 只要 Rust 服务处于“控制功能已开启”状态，手机即可随时发起连接。

## 4. 安卓客户端设计

### 4.1 结构与形态

- 使用原生 Android（Kotlin/Java），方便集成 `MediaCodec` 和底层输入处理。
- 主界面：
  - 设备列表（已配对电脑）。
  - 扫描二维码按钮（添加新电脑）。
- 游戏控制界面：
  - 全屏显示远程画面（固定横屏）。
  - 右侧边缘区域作为手势触发区，用于呼出导航栏。

### 4.2 视频接收与渲染

- 建立与 PC 的 UDP 视频流会话：
  - 握手后开始接收视频包。
  - 按帧序号重组为完整编码帧，喂给 `MediaCodec` 解码器。
- 渲染：
  - 使用 SurfaceView/TextureView 作为 `MediaCodec` 输出 surface。
  - 设定合适的渲染模式以减少缓冲（尽量低帧队列）。

### 4.3 输入与手势

- 多点触控：
  - 捕获屏幕上的多指触控事件（down/move/up），记录 pointer id 和坐标。
  - 将事件打包为自定义输入消息，通过 UDP（或可靠通道）发送至 PC。
- 右侧导航栏：
  - 在屏幕右侧留出一条窄边缘区域：
    - 从右向左滑动时，拦截手势并弹出垂直导航栏。
  - 导航栏按钮：
    - 返回
    - 主页
    - 多任务
    - 全局设置
    - 断开连接
- 系统返回键处理：
  - 在游戏控制界面内拦截系统返回键事件，不直接退出应用。
  - 将返回手势映射为“向 PC 发送 Android 返回键”输入。

### 4.4 全局设置菜单

- 在右侧导航栏中提供“设置”按钮，点击后弹出设置面板：
  - 分辨率设置：
    - 保持模拟器默认。
    - 跟随模拟器当前分辨率（只读显示）。
    - 预设分辨率（720p/1080p/2K）发送给 PC 端请求调整编码分辨率。（真正是否改变 MuMu 内部分辨率可选：初版先只调整编码输出尺寸。）
  - 刷新率设置：
    - 60/75/90fps 选项，发送到 PC 调整采集帧率。
  - 连接相关：
    - 显示当前电脑名称/IP。
    - “断开连接”按钮：只断开当前会话，保留配对关系。

## 5. 配对与连接流程细节

1. 用户在 PC 上启动 Rust 桌面程序，点击“开启控制功能”。
2. 程序：
   - 查找并绑定 MuMu 窗口。
   - 启动局域网监听端口。
   - 生成配对 token + IP/端口，展示二维码。
3. 手机首次使用：
   - 打开 App，选择“扫描二维码”。
   - 扫码后自动向 PC 发起配对请求。
   - 成功后将该 PC 记录到本地列表。
4. 以后使用：
   - 打开 App，在设备列表中选择该 PC → 连接。
   - PC 端同意/自动接受连接后开始推流和接收输入。
   - 用户在右侧边缘滑动唤出导航栏进行返回/主页/多任务等操作。
   - 点击“断开连接”按钮结束当前会话，但保留该 PC 记录。

## 6. 性能与延迟目标

- 端到端延迟目标：
  - 采集 + 编码：5–10ms（依赖 GPU 能力和编码参数）。
  - 网络传输（局域网）：1–5ms。
  - 解码 + 渲染：5–10ms。
  - adb 注入和游戏响应：若干毫秒级。
- 总体目标：理想条件下在 20ms 以内，常见条件下控制在 20–35ms 左右。
- 优化方向：
  - 画质与延迟的权衡通过设置菜单由用户控制（分辨率、帧率、码率）。
  - 默认尽量偏向低延迟配置。

## 7. 安全与稳定性

- 只在局域网内通信，不做公网穿透。
- 通过配对 token 和设备 ID 防止陌生设备随意连接。
- 对异常情况的处理：
  - 网络断开：PC 端停止推流，安卓端提示“连接中断”，允许重连。
  - MuMu 关闭：PC 端通知手机后停止推流，提示用户重新打开 MuMu。 

## 8. 后续可能扩展

- 增加简单录屏/截图功能（在 PC 端记录编码前的画面）。
- 支持多实例 MuMu，允许切换窗口。
- 支持其他模拟器，通过增加适配层（不同的窗口查找逻辑和 adb 配置）。
